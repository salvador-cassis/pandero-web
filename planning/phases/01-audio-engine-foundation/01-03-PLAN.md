From unmute-ios-audio v3.3.0 (via importmap):
```javascript
import unmuteAudio from 'unmute-ios-audio';

// Call ONCE, synchronously, before any user gesture.
// Registers listeners for: click, touchend, touchstart.
// On first activation: plays a 7-byte silent WAV via <audio> + silent Web Audio buffer.
// Routes Web Audio through the media channel (not ringer channel).
// Cleans up event listeners after success. No callback needed.
unmuteAudio();
```

iOS 'interrupted' state (after screen lock or backgrounding):
```javascript
// Must check BOTH states — 'interrupted' is iOS-only, 'suspended' is cross-browser
if (audioCtx.state === 'suspended' || audioCtx.state === 'interrupted') {
  await audioCtx.resume();
}
```

iOS Safari importmap support:
- Safari 16.4+ supports importmap
- iOS < 16.4 will fail silently on the module import
- If tester's iPhone is on iOS < 16.4: fallback is to bundle locally (not in scope for this plan — flag as open question)
