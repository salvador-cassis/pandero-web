<objective>
Patch the local SoundTouch processor to accept WSOLA tuning parameters and insert a GainNode into the audio graph, eliminating the robotic percussion artifacts from Phase 1 and preparing the signal chain for volume control.

Purpose: The Phase 1 PoC sounds robotic because SoundTouch's auto-calculated WSOLA sequence length (~110ms) is too coarse for percussion. Fixing this is a prerequisite to useful controls -- controls wired to a bad-sounding engine are not acceptable. The GainNode insertion is done here because it changes the audio graph topology (stNode -> gainNode -> destination instead of stNode -> destination), and all subsequent plans must wire against this new topology.

Output: Patched `soundtouch-processor.js`, refactored `poc.js` with GainNode and processorOptions.
</objective>

Current constructor (line 875):

constructor() {

Replace with:
```javascript
constructor(options) {
  super(options);
  this._pipe = new SoundTouch();
  this._samples = new Float32Array(256);
  this._outputSamples = new Float32Array(256);
  const po = (options && options.processorOptions) || {};
  if (po.sequenceMs || po.seekWindowMs || po.overlapMs) {
    this._pipe.stretch.setParameters(
      sampleRate,
      po.sequenceMs   || 0,
      po.seekWindowMs || 0,
      po.overlapMs    || 8
    );
  }
}
```

Key details:
- `sampleRate` is a global in `AudioWorkletGlobalScope` — use it directly, not `this.sampleRate` or `options.sampleRate`
- `this._pipe.stretch` is the `Stretch` instance (created at line 740 inside `SoundTouch` constructor)
- `setParameters(sampleRate, sequenceMs, seekWindowMs, overlapMs)` is defined at line 538 of the same file
- Values of 0 for sequenceMs/seekWindowMs tell `setParameters` to keep auto-calculation (the method checks `if (sequenceMs > 0)` before overriding)
- `overlapMs` defaults to 8 which matches the library's own default

Do NOT modify any other part of the file. Only the constructor changes.
