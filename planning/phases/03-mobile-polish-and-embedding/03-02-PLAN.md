  <action>
Create `pandero/player.js` as a complete ES module. The file has three sections: (A) guards and imports, (B) DOM creation and mounting, (C) audio engine + event wiring. Below is the exact structure.

**SECTION A: Guards and imports (top of file)**

1. Import unmute-ios-audio via DIRECT CDN URL (not bare specifier — eliminates importmap dependency per UI-03). This MUST be the very first line since ES module imports are hoisted:
```javascript
import unmuteAudio from 'https://esm.sh/unmute-ios-audio@3.3.0';
```

2. isSecureContext guard — must run before DOM creation:
```javascript
if (!window.isSecureContext) {
  const warn = document.createElement('p');
  warn.style.cssText = 'color:red;font-weight:bold;font-family:sans-serif';
  warn.textContent = '[Pandero] Must be served via HTTPS or localhost — not file://';
  document.getElementById('pandero-player')?.appendChild(warn);
  throw new Error('[pandero] Not a Secure Context. AudioWorklet will fail.');
}
```

3. Call `unmuteAudio()` immediately at module level (before any user gesture — same as poc.js).

4. Pre-fetch MP3 at module level (same as poc.js):
```javascript
const mp3Promise = fetch(new URL('./pandero.mp3', import.meta.url)).then(r => r.arrayBuffer());
```
Note: Use `import.meta.url` to resolve relative to player.js location, not the host page URL. This is critical for embedding — the host page may be at `/` while player.js is at `/pandero/player.js`.

**SECTION B: DOM creation and mounting**

5. Helper functions — `el(tag, className)` and `sliderRow(id, label, min, max, step, value, valId, valText)` exactly as shown in RESEARCH.md Pattern 2. The `sliderRow` function creates:
   - A `.pandero-slider-row` div containing:
     - A `.pandero-slider-header` div with a `<label for="pandero-{id}">` and optionally a `<span id="{valId}" class="pandero-value">` with valText
     - An `<input type="range" id="pandero-{id}" class="pandero-slider">` with the specified min/max/step/value

6. `buildDOM()` function — creates the entire widget DOM tree:
   - Root: `div.pandero-widget`
   - Hexagon button: `button.pandero-play-btn` with `type="button"`, `aria-label="Reproducir / Detener"`, textContent = `'\u25B6'` (▶)
   - Controls container: `div.pandero-controls` containing three slider rows:
     - `sliderRow('tempo', 'Tempo', 0.5, 1.5, 0.01, 1.0, 'pandero-tempo-val', '111 BPM')` (per D-10: first)
     - `sliderRow('pitch', 'Pitch', -6, 6, 1, 0, 'pandero-pitch-val', '0 st')` (per D-10: second)
     - `sliderRow('volume', 'Volumen', 0, 1, 0.01, 1.0, null, null)` (per D-10: third, no value display)
   - NO reset button (per D-13)
   - Returns root element

7. Mount point discovery and DOM injection:
```javascript
const mount = document.getElementById('pandero-player');
if (!mount) throw new Error('[pandero] Mount point #pandero-player not found');
const root = buildDOM();
mount.appendChild(root);
```

**SECTION C: Audio engine (migrated from poc.js exactly) + event wiring**

8. Module-level singletons — copy exactly from poc.js lines 17-25:
   `audioCtx`, `stNode`, `gainNode`, `source`, `audioBuffer`, `isInitialized`, `isPlaying`

await audioCtx.audioWorklet.addModule(new URL('./soundtouch-processor.js', import.meta.url));

    - `document.getElementById('pitch')` → `document.getElementById('pandero-pitch')`
    - Keep ALL WSOLA processorOptions: `{ sequenceMs: 40, seekWindowMs: 15, overlapMs: 8 }`
    - Keep tandem pattern: BOTH `source.playbackRate.value = ratio` AND `stNode.parameters.get('playbackRate').value = ratio`

11. `togglePlayback()` function — copy from poc.js lines 102-124. Changes:
    - Reference the hexagon button via stored variable (from buildDOM), not `document.getElementById('toggle-btn')`
    - Remove the `status` element references (no status paragraph in widget)
    - Instead of updating `status.textContent = 'Cargando...'`, disable the button: `hexBtn.disabled = true;` before init(), then `hexBtn.disabled = false;` after
    - Keep the same icon symbols as poc.js: ▶ (`'\u25B6'`) when stopped, ⏸ (`'\u23F8'`) when playing. The ACTION is stop (source.stop()), but the ICON shows ⏸ — this matches poc.js exactly.

12. `handleTempoChange(e)` — copy from poc.js lines 131-137. Change: `document.getElementById('tempo-val')` → `document.getElementById('pandero-tempo-val')`. Keep BPM formula: `Math.round(ratio * 111)`.

13. `handlePitchChange(e)` — copy from poc.js lines 143-148. Change: `document.getElementById('pitch-val')` → `document.getElementById('pandero-pitch-val')`. Keep signed display format.

14. `handleVolumeChange(e)` — copy exactly from poc.js lines 154-157. No element references to change.

15. `handleReset()` — DO NOT include. Per D-13, no reset button in the widget.

16. Event wiring — wire to the programmatically created elements. Store references from buildDOM() or query by pandero-* IDs:
```javascript
const hexBtn = root.querySelector('.pandero-play-btn');
const tempoSlider = document.getElementById('pandero-tempo');
const pitchSlider = document.getElementById('pandero-pitch');
const volumeSlider = document.getElementById('pandero-volume');

hexBtn.addEventListener('click', togglePlayback);
tempoSlider.addEventListener('input', handleTempoChange);
pitchSlider.addEventListener('input', handlePitchChange);
volumeSlider.addEventListener('input', handleVolumeChange);
```

17. Space key handler — copy from poc.js lines 185-190. Keep exactly as-is (document-level keydown).

**What NOT to include:**
- No `handleReset()` function (D-13)
- No `reset-btn` event listener (D-13)
- No `status` element (no loading text — just button disable/enable)
- No importmap dependency (UI-03 — direct CDN URL instead)
- No `document.write` (replaced by createElement guard)

**Scope isolation note (UI-04):**
- The only `window.*` reference allowed is `window.isSecureContext` in the guard (read-only check, not an assignment).
- No `window.foo = bar` assignments anywhere.
- `type="module"` provides automatic strict-mode scope isolation.

**CRITICAL audio rules from CLAUDE.md (non-negotiable):**
- AudioContext created ONLY inside user gesture (click handler via togglePlayback)
- Tandem pattern: `source.playbackRate.value = ratio` AND `stNode.parameters.get('playbackRate').value = ratio` — never use `stNode.tempo`
- Single AudioContext per widget (singleton pattern)
- `source.loop = true` for continuous playback
- GainNode created once in init(), persists across restarts
